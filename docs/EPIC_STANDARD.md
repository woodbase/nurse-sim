Epics define architectural domains.

Each Epic must represent:
- One system
OR
- One major procedure module

Epics must not mix responsibilities.

---

## Example Epics

- Epic: Interaction System
- Epic: Procedure System
- Epic: Scoring & Assessment
- Epic: PVK Procedure
- Epic: Catheterization Procedure

---

## Epic Requirements

Each Epic must contain:

- Clear system boundary
- Folder ownership
- Communication expectations
- Scalability note

---

## AI Implementation Rule

If issue has Epic label:
- Respect architectural boundary.
- Do not modify other systems unless explicitly stated.
